# 백구(백세까지 구인구직) — 제품 요구사항 정의서(PRD)

**문서 목적:** 직업정보제공(Market플레이스) MVP 범위, 법·보안, 기술·UX 원칙, 우선순위를 한곳에 고정한다.  
**버전:** 1.2  
**기준일:** 2026-04-13

> **작업 시 참고:** 아래 **§15 구현 현황**에서 `[미구현]` 항목은 요구사항에서 삭제된 것이 아니라 **아직 코드에 없거나 미연동**인 것이다. 백로그·스프린트 계획 시 누락 방지용으로 유지한다.

## 1. 제품 개요

### 1.1 한 줄 정의

**백구**는 고용주와 시니어 구직자 사이에 구인·구직 **정보가 오가는 장(Marketplace)**을 제공하는 **직업정보제공** 서비스다. 직접 고용·알선이 아니라 **정보 제공·지원**에 초점을 둔다.

### 1.2 서비스명·포지셔닝

| 항목 | 내용 |
|------|------|
| 서비스명 | 백구 (백세까지 구인구직) |
| 플랫폼 유형 | 직업정보제공사업 (알바천국/알바몬 유사) |
| 디자인 컨셉 | Apple 스타일 미니멀리즘 — 큰 글씨, 고대비, 단순 정보 구조, 직관적 아이콘 |

### 1.3 제품 원칙 (비기능 요구사항과 연결)

- **신뢰가 자본:** 기술 과시보다 투명한 인증·마스킹·용어 통제.
- **시니어 우선 접근성:** 터치 영역·대비·오클릭 방지가 기능과 동등한 우선순위.
- **1인 개발 현실성:** 오버엔지니어링 금지, 빠른 배포·측정·수정 루프.

## 5. MVP 범위 & 우선순위

**기술 스택 (고정):** Next.js(App Router) · Supabase(Auth/DB/Realtime 검토) · Tailwind CSS · Vercel

| 순서 | 영역 | MVP 내용 |
|:----:|------|----------|
| 1 | **Home** | 내 주변 일자리 **지도**(Map API), 핀 과다 시 **클러스터링** 설계 반영 |
| 2 | **Resume** | 아이콘 기반 시니어 **간편 프로필/이력** |
| 3 | **Employer** | 사업자 번호 유효성 검사 + **인건비·혜택 계산기**(시급 입력 시 실질 부담액 강조 등) |
| 4 | **Admin** | 부적절 공고 필터링(OpenAI) — **cron 배치(예: 10분)** 또는 **의심 키워드 선별** 후 검사 |

### 5.1 Post-MVP / 확장 (문서화만 — 스코프 고정용)

- Next.js **ISR**로 공고 리스트·랜딩 성능
- Supabase **Realtime**: 고용주 이력서 열람 시 시니어 알림 등
- 지도 **마커 클러스터링** 고도화, Map/OpenAI **비용 모니터링**

## 6. 기능 상세 요구사항

### 6.1 홈 / 지도

- 사용자 위치 기반 목록·지도; **도보권 우선** 정렬/필터.
- 권한 거부 시 **행정동「우리 동네 설정」**으로 대체 플로우 필수.
- 성능: 초기에는 단순 구현, 핀 밀도 증가 시 **클러스터링** 적용.
- **개발 단계:** 실사용자 데이터 전 **`Faker.js` 등으로 가짜 공고·좌표를 대량 생성**해 지도에 핀 수백 개를 찍고, 클러스터링·성능 감각을 먼저 확보한다(§14.3 참고).

### 6.2 간편 이력서(Resume)

- 타이핑 부담 최소화: 아이콘 선택, 단계형 UI, 음성 입력(단계적 도입 가능).
- 고용주에게 내려주는 데이터는 **채용 확정 전 마스킹 규칙** 준수.

### 6.3 고용주 — 인증 & 계산기

- **사업자 인증:** 공고 최초 등록 시 1회, `is_verified` 등 플래그 저장.
- **인건비 계산기:** 클라이언트 실시간 계산 UI; **지원금 반영 후 실질 부담**을 시각적으로 강조.
- **정부 지원금·혜택 계산 로직**은 정책 변경에 대비해 UI와 분리한다. 예: `lib/calculator.ts`에만 산식·상수·버전 코멘트를 두고, 화면은 해당 모듈을 호출만 한다(§14.2).

### 6.4 관리자 — 공고 검수

- OpenAI 호출 **전량 실시간 금지**(비용); 배치 또는 키워드 트리거.
- 검수 결과·사유 로그(감사·재처리용 필드는 스키마에 여지).

### 6.5 알림

- 카카오 알림 우선 전략; 구현 시 **알림톡 API + Supabase Edge Functions** 등 서버리스 연동 검토.
- (선택) 이력서 열람 실시간 알림 — Realtime 연동 시 UX 효과 기대.

## 14. 개발 규약 (Cursor·코드 구조)

### 14.1 `.cursorrules` 적극 활용

프로젝트 루트에 `.cursorrules` 파일을 두고, AI·협업 시 일관된 UI/스택을 강제한다. 예시 문구:

> 모든 컴포넌트는 `lucide-react` 아이콘을 사용하고, `Radix UI` 또는 `Shadcn UI` 기반의 미니멀한 스타일로 작성해줘. 시니어 타겟이므로 폰트 크기는 기본 `18px` 이상, 버튼 높이는 `60px`로 고정해.

- 위 규약은 §7 UX 표와 정합을 맞출 것(아이콘·터치·타이포).

### 14.2 정부 지원금 로직의 모듈 분리

- 고령 채용 지원금·인턴십 등 **제도는 수시로 개정**되므로, 산식·적용 조건·금액 상수는 **단일 모듈**에만 둔다.
- 권장: `lib/calculator.ts`(또는 동등 경로)에 **계산 로직만** 모으고, 컴포넌트·API 라우트는 이 모듈을 import한다.
- 법·고시 변경 시 해당 파일(및 테스트)만 수정하면 되도록 **주석에 적용 기준일·출처(예: 고시 번호)**를 남긴다.

### 14.3 가짜 데이터로 먼저 검증

- 실사용자가 쌓이기 전까지는 지도·리스트 UX·성능 검증이 어렵다. **`@faker-js/faker` 등으로 공고·좌표·카테고리를 대량 생성**해 개발·스테이징에서 사용한다.
- 목표: 지도에 **핀이 수백 개** 있는 상태에서 렌더링·**마커 클러스터링**·스크롤·필터 동작을 미리 확인한다.
- 프로덕션 빌드에는 포함하지 않거나, 환경 변수로만 활성화한다.

## 부록: 스택·확장성·운영 (참고)

### A. 인프라 (Vercel & Next.js)

- Auto-scaling 서버리스, Edge로 정적·반정적 데이터 전달 가속.

### B. 데이터 (Supabase)

- PostgreSQL, Connection Pooling으로 트래픽 급증 시 연결 안정성.

### C. 1인 개발 운영

- 자동화(인증·검수), 커뮤니티·AI는 개발 보조에만, **비즈니스 판단은 현장 피드백** 우선.
- 단계적 확장: 전국보다 **특정 구·시 단위 1위** 전략.

### D. Cursor / Vibe Coding

- 상세 규약은 **§14** 참고(`.cursorrules` 문구, `lib/calculator.ts`, Faker 기반 시드).
- 도메인 폴더 구조(Senior, Employer) 유지로 AI 맥락 유지 용이.

### 15.1 인증·프로필

| 요구 | 상태 | 비고 |
|------|------|------|
